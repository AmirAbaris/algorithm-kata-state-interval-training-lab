# When Stuck — 10 Minute Protocol

Confusion ≠ inability. It usually means structure isn't on paper yet.

## Minute 0–3: Stop coding

Close `solution.go`. Re-read only:

- **Input / Output** sections in `problem.md`
- The **smallest** failing test name

## Minute 3–7: State table

Fill one row in `PROBLEM_READING.md` for the failing test case.

Ask aloud:

- "What is **current** right now?"
- "What am I **keeping** from previous steps?"
- "What **changes** after this step?"

## Minute 7–10: Shrink

- Can you hard-code the return for that one test?
- Can you solve it for length-1 input only?
- For intervals: draw a number line.

## If still stuck

1. Re-read the **Connection** section in `problem.md` (links to previous track).
2. Open the **previous** kata's solution if you finished it — compare pattern only.
3. Take a 15-minute break. Come back to the state table, not the editor.

## Do not

- Jump to a later track "for motivation"
- Rewrite the whole function hoping it works
- Interpret stuck as "I'm bad at algorithms"

You are building **tolerance for understanding**, not speed.
